## Critical Issues

### [C1] `vdem_legislative_corruption` comparison labels are swapped

- **File:** `scripts/score_countries.py:271`
- **Bug:** The `comparison_label` is `["Most corrupt legislature among", "Least corrupt legislature among"]`. The first label maps to the country with the highest raw `v2lgcrrpt` value. But higher `v2lgcrrpt` = LESS corrupt (inverted variable, confirmed by V-Dem codebook: Luxembourg at 3.50 = "essentially no legislative corruption"). So the label says "Most corrupt legislature among [peer group]" for the country with the *cleanest* legislature.
- **Impact:** Every country's legislative corruption peer comparison text in the UI is semantically inverted. Luxembourg would display "Most corrupt legislature among Western Europe" when it has the least corrupt legislature.
- **Suggested fix:** Swap the labels:
  ```python
  "comparison_label": ["Least corrupt legislature among", "Most corrupt legislature among"],
  ```
- **Confidence:** High
- **Found by:** Logic & Correctness, Contract & Integration

### [C2] `methodology_id` selection via `unique()[-1]` may select stale data

- **File:** `scripts/score_countries.py:1204` (also line 1174)
- **Bug:** Both `load_fsi_data()` and `load_fsi_related_jurisdictions()` select the "latest" FSI methodology using `df["methodology_id"].unique()[-1]`. Pandas `unique()` preserves insertion order (it does NOT sort). In the real production CSV, the naming convention changed from `fsi2022` to `fsi_8.0`. If `fsi_8.0` rows appear before `fsi2022` rows in the CSV (as they do in the actual production data), `unique()[-1]` returns `fsi2022` — the **older** edition.
- **Impact:** The entire TF domain could be scored using 3-year-old data instead of the current edition. The test fixture happens to have the "right" order (fsi2022 before fsi2024), masking the bug.
- **Suggested fix:** Use explicit recency ordering. If the CSV has a `scoring_timestamp` column:
  ```python
  latest_scoring = df.sort_values("scoring_timestamp").iloc[-1]["methodology_id"]
  ```
  Or at minimum use `sorted()[-1]` for lexicographic ordering (works for current naming conventions).
- **Confidence:** High
- **Found by:** Logic & Correctness, Error Handling & Edge Cases

- **File:** `scripts/score_countries.py:1513-1545`

- **Confidence:** High
- **Found by:** Concurrency & State

### [I3] Test fixture CSV schema diverges from real FSI data

- **File:** `tests/python/fixtures/fsi/fsi_jurisdictions.csv`
- **Bug:** The test fixture has columns `jurisdiction_id,jurisdiction_name,methodology_id,index_score,index_value,global_scale_weight`. The real production CSV has `scoring_timestamp,methodology_id,...,index_score,index_value,index_share,rank,gsw`. Key differences: (1) fixture lacks `index_share`, so `load_fsi_related_jurisdictions()` will never populate `fsi_share_pct` in tests, producing "0.0% of global financial secrecy value" for GBR; (2) fixture lacks `scoring_timestamp` needed for robust methodology selection; (3) fixture's methodology_id naming matches different conventions than production.
- **Impact:** Tests cannot catch regressions in related-jurisdictions logic or methodology selection. The `unique()[-1]` bug (C2) works "correctly" in tests but fails in production.
- **Suggested fix:** Add `index_share` and `scoring_timestamp` columns to the fixture, and ensure methodology_id values exercise real selection logic.
- **Confidence:** High
- **Found by:** Logic & Correctness, Error Handling & Edge Cases, Contract & Integration
