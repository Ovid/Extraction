## Important Issues

### [I1] Two World Bank indicators fetched but never used in scoring pipeline
- **File:** `scripts/fetchers/worldbank.py:44-45`
- **Bug:** The worldbank fetcher still downloads `wb_wgi_reg_quality` (RQ.EST) and `wb_wgi_gov_effectiveness` (GE.EST), but `INDICATOR_CONFIG` in `score_countries.py` no longer references either CSV. These indicators were previously used and their `SOURCE_URLS` entries were removed from `app.js`, but the fetcher was not updated.
- **Impact:** Wasted API calls on every `fetch_all.py` run. More importantly, the fetcher and scorer are out of sync about which indicators are part of the project, creating confusion for future contributors.
- **Suggested fix:** Remove the two entries from `INDICATORS` in `worldbank.py`, or re-add them to `INDICATOR_CONFIG` if the removal was accidental.
- **Confidence:** High
- **Found by:** Contract & Integration

### [I2] `restructure_institutional_gatekeeping` silently skips when only one sub-group has data
- **File:** `scripts/score_countries.py:1563`
- **Bug:** The condition `if quality_scores and purpose_scores:` requires BOTH quality (corruption control + rule of law) and purpose (egalitarian + participatory) sub-groups to have data. If a country has WB corruption data but no V-Dem egalitarian/participatory data, restructuring is silently skipped and the domain keeps its flat-average score. There is no logging, no justification_detail note, and no output flag.
- **Impact:** Countries with partial V-Dem coverage receive a different weighting methodology than countries with full coverage, with no indication in the output. This undermines the stated goal of giving equal weight to "how well do institutions function?" vs. "who do institutions serve?"
- **Suggested fix:** At minimum, add a justification_detail note when restructuring is skipped due to partial data. Optionally, use whichever sub-group IS available as the domain score with a confidence downgrade.
- **Confidence:** High
- **Found by:** Logic & Correctness, Error Handling & Edge Cases

- **[S5]** `js/app.js:456-459` — `esc(null)` returns the literal string `"null"`. Adding `if (s == null) return '';` would prevent displaying "null" if upstream data is malformed. Very low risk given current call sites.
