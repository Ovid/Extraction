## Important Issues

### [I1] Duplicate NUMERIC_MAP entry 699 -> IND breaks picker-to-map selection for India

- **File:** `js/lib.js:123`
- **Bug:** Both `356: 'IND'` (line 69, correct ISO 3166-1 numeric) and `699: 'IND'` (line 123, unassigned/deleted ISO code) map to India. The picker reverse-lookup at `app.js:742` (`Object.entries(numericToAlpha3).find(([, a3]) => a3 === code)`) returns the first match. If `699` is returned instead of `356`, `selectCountry(code, "699")` attempts to highlight a TopoJSON feature with `d.id === 699`, which doesn't exist. India won't be highlighted or zoomed on the map when selected from the picker.
- **Impact:** User-visible: clicking India in the country picker may fail to zoom/highlight India on the map. The data panel displays correctly (uses alpha3), but the visual map feedback is missing.
- **Suggested fix:** Remove `699: 'IND'` from NUMERIC_MAP.
- **Confidence:** High
- **Found by:** Logic & Correctness, Error Handling & Edge Cases, Concurrency & State (3 specialists)

### [I2] `_load_fsi_csv()` accesses `methodology_id` column without existence check

- **File:** `scripts/score_countries.py:1181,1188,1190`
- **Bug:** The function defensively checks `if "scoring_timestamp" in df.columns` (line 1178) but accesses `df["methodology_id"]` unconditionally on lines 1181, 1188, and 1190. If the FSI CSV lacks a `methodology_id` column (e.g., TJN schema change), this raises an unhandled `KeyError` that crashes the entire scoring pipeline. Both `load_fsi_data()` and `load_fsi_related_jurisdictions()` call this function.
- **Impact:** Scoring pipeline crash on FSI schema changes, rather than graceful degradation to empty results.
- **Suggested fix:** Add early guard: `if "methodology_id" not in df.columns: return pd.DataFrame(), None`
- **Confidence:** High
- **Found by:** Error Handling & Edge Cases

### [I3] Map zoom and panel state diverge for no-data territory clicks

- **File:** `js/app.js:262-299`
- **Bug:** In `selectCountry()`, the zoom logic (lines 273-285) runs whenever `numericId != null || alpha3` is truthy and a matching SVG element is found. This executes BEFORE `getCountryData(alpha3)` checks for data at line 288. For territories with no scores.json entry, the map zooms and highlights the territory, but the panel shows "No data for X" and `selectedCountryCode` is set to null. The map remains zoomed to a country shown as unselected, with no programmatic way to reset except the zoom-reset button.
- **Impact:** User-visible: after clicking a no-data territory, the map is zoomed to it but the panel says nothing is selected. Weight adjustments won't update the panel because `selectedCountryCode` is null.
- **Suggested fix:** Either skip the zoom when there's no country data, or reset the zoom when clearing the selection.
- **Confidence:** Medium
- **Found by:** Concurrency & State

- `js/app.js:508` -- `conf` value used in innerHTML without `esc()`: `Confidence: ${conf.replace('_', ' ')}`. Inconsistent with other escaped values. Data is from static JSON, so not practically exploitable, but should use `esc()` for consistency. (Security)

- `index.html` -- No Content-Security-Policy meta tag. The app uses innerHTML in several places. A CSP would limit impact of any future XSS. Low priority for a static local site. (Security)

- `js/app.js:505` -- `related_jurisdictions` array (per-territory secrecy scores and FSI share percentages) is stored in scores.json but only the `related_jurisdictions_note` summary text is rendered. The structured data could provide useful drill-down detail. (Plan Alignment)
